# Jujutsu Describe Command

現在の変更にメッセージを追加します（jjでは変更は自動的に記録されるため、明示的なコミットは不要）。

## 手順

1. `jj status` で現在の変更を確認
2. `jj diff` で変更内容を確認
3. `jj log -r @` で現在のリビジョンを確認
4. 変更内容を分析し、適切なメッセージを作成
5. `jj describe -m "メッセージ"` でメッセージを設定
6. `jj new` で新しい作業リビジョンを作成（次の作業に備える）

## コミットメッセージのルール

- 日本語で記述
- 変更の「何を」ではなく「なぜ」に焦点を当てる
- 1-2文で簡潔に
- 末尾に以下を追加:

```
Co-Authored-By: Claude Opus 4.5 <noreply@anthropic.com>
```

## 注意事項

- `.env` や認証情報を含むファイルはコミットしない
- ユーザーが明示的に要求しない限りプッシュしない
- jjでは変更が自動的にスナップショットされるため、`add`は不要
